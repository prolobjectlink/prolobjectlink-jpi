28  /**
29   * Prolog Provider is the class to interact with all prolog components (data
30   * types, constants, logger, parser, converter and engine). Allow create data
31   * types like atoms, numbers, lists, structures and variable terms.
32   * 
33   * Allow interact with the under-lying prolog engine to realize inferences
34   * operations. Other used components are the prolog parser to parse prolog terms
35   * in string form. The Prolog Provider offer a prolog system logger to report
36   * every errors or exceptions.
37   * 
38   * @author Jose Zalacain
39   * @since 1.0
40   */

368 	/**
369 	 * Create a prolog structure that represent an expression defined by your left
370 	 * and right operands separated by infix operator. The structure instance have
371 	 * like functor the expression operator and have two operands arguments terms.
372 	 * In other words the indicator for the resulting instance term is
373 	 * <tt>operator/2</tt>. The term creation not check operator definition and for
374 	 * this reason during inference process if the operator is not a supported
375 	 * built-in or define by <tt>op/3</tt> the inference fail.
376 	 * 
377 	 * <pre>
378 	 * PrologVariable x = provider.newVariable("X", 0);
379 	 * PrologDouble pi = provider.newDouble(Math.PI);
380 	 * PrologStructure plusExp = provider.newStructure(x, "+", pi);
381 	 * System.out.println(plusExp);
382 	 * </pre>
383 	 * 
384 	 * @param left     left hand operand
385 	 * @param operator infix operand
386 	 * @param right    right hand operand
387 	 * @return a prolog structure that represent an expression
388 	 * @since 1.0
389 	 */
