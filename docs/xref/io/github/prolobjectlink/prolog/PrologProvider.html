32  /**
33   * Prolog Provider is the class to interact with all prolog components (data
34   * types, constants, logger, parser, converter and engine). Allow create data
35   * types like atoms, numbers, lists, structures and variable terms.
36   * 
37   * Allow interact with the under-lying prolog engine to realize inferences
38   * operations. Other used components are the prolog parser to parse prolog terms
39   * in string form. The Prolog Provider offer a prolog system logger to report
40   * every errors or exceptions.
41   * 
42   * @author Jose Zalacain
43   * @since 1.0
44   */
45  public interface PrologProvider extends PrologParser, Map<Class<?>, Prologable<?>> {

373 	 * Create a prolog structure that represent an expression defined by your left
374 	 * and right operands separated by infix operator. The structure instance have
375 	 * like functor the expression operator and have two operands arguments terms.
376 	 * In other words the indicator for the resulting instance term is
377 	 * <tt>operator/2</tt>. The term creation not check operator definition and for
378 	 * this reason during inference process if the operator is not a supported
379 	 * built-in or define by <tt>op/3</tt> the inference fail.
380 	 * 
381 	 * <pre>
382 	 * PrologVariable x = provider.newVariable("X", 0);
383 	 * PrologDouble pi = provider.newDouble(Math.PI);
384 	 * PrologStructure plusExp = provider.newStructure(x, "+", pi);
385 	 * System.out.println(plusExp);
386 	 * </pre>
387 	 * 
388 	 * @param left     left hand operand
389 	 * @param operator infix operand
390 	 * @param right    right hand operand
391 	 * @return a prolog structure that represent an expression
392 	 * @since 1.0
393 	 */
394 	public PrologTerm newStructure(PrologTerm left, String operator, PrologTerm right);
