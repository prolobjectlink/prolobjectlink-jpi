35  /**
36   * Prolog thread is a thread of execution in a program. The Java Virtual Machine
37   * allows an application to have multiple threads of execution running
38   * concurrently.
39   * 
40   * Every thread has a priority. Threads with higher priority are executed in
41   * preference to threads with lower priority. Each thread may or may not also be
42   * marked as a daemon. When code running in some thread creates a new Thread
43   * object, the new thread has its priority initially set equal to the priority
44   * of the creating thread, and is a daemon thread if and only if the creating
45   * thread is a daemon.
46   * 
47   * @author Jose Zalacain
48   * @since 1.1
49   */
