37  /**
38   * <p>
39   * Ancestor prolog data type. All Prolog data types {@link PrologAtom},
40   * {@link PrologNumber}, {@link PrologList}, {@link PrologStructure} and
41   * {@link PrologVariable} are derived from this class. All before mentioned
42   * classes extends from this class the commons responsibilities. Extends from
43   * {@link Comparable} to compare the current term with another term based on
44   * Standard Order.
45   * </p>
46   * 
47   * <p>
48   * The Prolog Provider is the mechanism to create a new Prolog terms invoking
49   * the correspondent methods. {@link PrologProvider#newAtom(String)} for atoms,
50   * {@link PrologProvider#newStructure(String, PrologTerm...)} for structures,
51   * {@link PrologProvider#newVariable(String, int)} for variables and so on.
52   * </p>
53   * 
54   * <p>
55   * With {@link #getType()} the term type can be retrieve. Every term have a type
56   * defined in {@link PrologTermType}. The prolog type have by definition one
57   * order (Standard Order) where Variables &lt; Atoms &lt; Numbers &lt;
58   * Compounds. This order is usable to compare two terms.
59   * </p>
60   * 
61   * <p>
62   * Prolog terms have two specials properties, the functor and the arity. Functor
63   * is the compound term or atom name and arity is the number of arguments
64   * present in the compound terms. Variable and Number don't have this
65   * properties. This properties are accessible from {@link #getArity()} and
66   * {@link #getFunctor()} methods. Variable and Number raise an
67   * {@link ArityError} and {@link FunctorError} respectively. Based on functor
68   * and arity atoms and compound terms have a predicate indicator that is the
69   * signature for the term. The term indicator is an string formed by the
70   * concatenation of {@link #getFunctor()} and {@link #getArity()} separated by
71   * slash.
72   * </p>
73   * 
74   * <p>
75   * The methods {@link #getArguments()} and {@link #getArgument(int)} allow
76   * retrieve the arguments and the some specific arguments for compounds terms
77   * respectively. For atomics terms {@link #getArguments()} return an empty term
78   * array and {@link #getArgument(int)} raise a
79   * {@link ArrayIndexOutOfBoundsException} because don't have arguments. For this
80   * cases is a good practice check compound terms before invoke this methods.
81   * </p>
82   * 
83   * <pre>
84   * if (t.isCompound()) {
85   * 	PrologTerm x = t.getArgument(i);
86   * 	System.out.println(x);
87   * }
88   * </pre>
89   * 
90   * Prolog unification is the process that involves one or more variables such
91   * that they are instantiated with the necessary terms in order to make two
92   * terms are identical. The prolog term interface provide
93   * {@link #unify(PrologTerm)} method to check that the current term unify with
94   * the given term. Prolog unification algorithm is based on three principals
95   * rules:
96   * <ul>
97   * <li>If x and y are atomics constants then x and y unify only if they are same
98   * object.</li>
99   * <li>If x is a variable and y is anything then they unify and x is
100  * instantiated to y. Conversely, if y is a variable then this is instantiated
101  * to x.</li>
102  * <li>If x and y are structured terms then unify only if they match (equals
103  * funtor and arity) and all their correspondents arguments unify.</li>
104  * </ul>
105  * 
106  * <pre>
107  * PrologAtom atom = provider.newAtom("smith");
108  * PrologAtom atom1 = provider.newAtom("doe");
109  * // true because the atoms are equals
110  * assertTrue(atom.unify(atom));
111  * // false because the atoms are different
112  * assertFalse(atom.unify(atom1));
113  * PrologVariable variable = provider.newVariable("X", 0);
114  * // true because atom and variable unify
115  * assertTrue(atom.unify(variable));
116  * </pre>
117  * 
118  * @author Jose Zalacain
119  * @since 1.0
120  */
